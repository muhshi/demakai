💡 Kesimpulan Keseluruhan & Rekomendasi

1. Perbandingan Metode Pencarian

  • 🔹 Metode "None" (Tanpa Preprocessing): SQL cenderung sangat ketat (exact match), sehingga keyword yang agak panjang atau memiliki variasi susunan kata mudah gagal (MRR rendah). Hybrid (Vector) sedikit lebih baik karena mampu menangkap kemiripan makna, namun masih kurang maksimal jika terdapat banyak kata hubung/stopword.
  • 🔹 Metode "Basic" (Case-fold, No-punct, Token-OR): Ini adalah sweet spot untuk SQL LIKE. Dengan memecah query menjadi token (OR), SQL berhasil menangkap dokumen yang setidaknya mengandung satu kata dari pencarian. Hal ini sangat menaikkan persentase sukses (Top 1 & Top 3). Untuk Hybrid, pemisahan tanda baca juga membuat ekstraksi embedding feature lebih bersih.
  • 🔹 Metode "Advanced" (Stemming & Stopword Removal): Sesuai temuan di tabel, seringkali metode Advanced justru menurunkan tingkat akurasi. Mengapa? Karena proses stemming (memotong imbuhan) sering mengubah bentuk kata dasar dan menghapus stopword (seperti "dan", "atau", "dari") membuat konteks kalimat terpotong. Untuk dataset formal seperti KBLI/KBJI, ketepatan susunan kata seringkali krusial, sehingga pemotongan ekstrem ini justru merugikan pemahaman semantik oleh model Hybrid.

2. Kesimpulan Utama

Kombinasi Hybrid Search + Preprocessing Basic terbukti sebagai kandidat terbaik yang konsisten. SQL LIKE + Basic memang tinggi akurasinya untuk data exact match, namun pada aplikasi nyata masyarakat sering menggunakan kosakata padanan/sinonim yang tidak persis ada di dalam database KBLI/KBJI (misal: "toko warung" atau "bengkel tambal"). Disinilah Vector Search (Hybrid) bekerja optimal apabila teks aslinya dibiarkan utuh tanpa di-stem terlalu ekstrem.

3. Saran untuk Pengembangan ke Depan (Next Steps)

  1. Nonaktifkan Advanced Preprocessing: Hindari stemming menggunakan sastrawi untuk pencarian ini, karena embedding model BERT cenderung lebih optimal membaca kalimat utuh (yang menyertakan imbuhan) untuk memahami konteks bahasa Indonesia.
  2. Perluas Data Ground Truth: Daftarkan lebih banyak contoh kalimat kueri natural/acak (yang biasa diketik user awam) ke dalam pengujian, bukan hanya kalimat murni KBLI, supaya efektivitas Hybrid Search (skor semantik) bisa terlihat jauh lebih dominan dibanding fungsionalitas SQL LIKE.
  3. Atur Ulang Bobot (Weighting) Hybrid: Bereksperimenlah kembali dengan rasio alpha (misalnya 70% vector, 30% keyword) di SearchService untuk mencari kombinasi yang bisa menekan error false positive ketika kemunculan kata tidak terlalu relevan.
