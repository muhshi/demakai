📊 Ringkasan Temuan

Metode terbaik berdasarkan MRR adalah: 1. SQL LIKE (Baseline) dengan MRR = 0.8861.

🔍 SQL LIKE Baseline
Tanpa preprocessing, SQL LIKE bekerja sangat buruk pada query informal berbasis bahasa lapangan. Hanya cocok jika kata dalam query persis sama dengan teks di database KBLI. MRR sangat rendah karena gap leksikal antara query pengguna dan deskripsi formal KBLI.
🧠 Dampak Hybrid Search
Penambahan semantic search (embedding Gemini) meningkatkan kemampuan sistem memahami makna di balik kata. Query seperti "mencuci motor" dapat dicocokkan secara konseptual dengan entri KBLI yang tidak mengandung kata tersebut secara harfiah. Namun tanpa preprocessing, Hybrid masih terbatas untuk query sangat informal.
✂ Dampak Preprocessing
Query Expansion adalah game-changer: satu query diperluas menjadi puluhan sinonim dan variasi. Ini meningkatkan recall secara dramatis — MRR SQL LIKE naik dari sangat rendah menjadi 0.8378. Menariknya, SQL LIKE + Preprocessing mengungguli Hybrid + Preprocessing karena expanded tokens sangat cocok untuk pencocokan leksikal langsung.
📋 Dampak Contoh Lapangan
Field contoh_lapangan berisi deskripsi aktivitas nyata dalam bahasa informal. Sistem sudah mencakup field ini di semua query SQL LIKE (via ILIKE contoh_lapangan) dan mendapat boost prioritas di Hybrid. Ini menjembatani gap bahasa antara query lapangan informal dan terminologi formal KBLI/KBJI.
🏆 SQL LIKE vs Hybrid (dengan Prep)
Dengan preprocessing, SQL LIKE (MRR=0.8378) lebih unggul dari Hybrid (MRR=0.7764). Hal ini terjadi karena Query Expansion menghasilkan token-token yang sangat spesifik, sehingga pencocokan leksikal SQL LIKE jauh lebih presisi. Semantic search bisa menambah noise karena embedding mencakup konteks yang terlalu luas.
✅ Rekomendasi Sistem Terbaik
Berdasarkan evaluasi ini, SQL LIKE + Preprocessing (+ Contoh Lapangan) adalah konfigurasi paling optimal untuk sistem DEMAKAI: sederhana (tidak butuh GPU/API embedding setiap query), lebih cepat, dan menghasilkan MRR tertinggi. Hybrid tetap direkomendasikan sebagai fallback untuk query abstrak tanpa keyword yang jelas.
