🔍 Sistem SQL LIKE + Preprocessing + Contoh Lapangan

Pada metode ini, query pengguna terlebih dahulu diproses menggunakan preprocessing (advanced + expansion sinonim), kemudian hasilnya digunakan untuk pencarian berbasis teks (SQL LIKE) pada dua field utama:

  • Deskripsi resmi KBLI/KBJI — teks formal dari klasifikasi baku
  • Contoh lapangan — deskripsi aktivitas nyata, bahasa informal, deskripsi pekerjaan di masyarakat

Perbedaan dari Baseline: Baseline SQL LIKE hanya mencari pada deskripsi formal tanpa preprocessing. Di sini, query diperluas dengan sinonim dan variasi, lalu dicari pada KBLI + contoh lapangan, sehingga jangkauan pencarian jauh lebih luas.

🔍 Preprocessing Meningkatkan Jangkauan
Dengan query expansion, satu query dapat diubah menjadi puluhan token dan sinonim. Contoh: "TUKANG BATU" diperluas menjadi "tukang, batu, teknisi, pekerja, operator". SQL LIKE kemudian mencari semua kemungkinan tersebut sehingga recall meningkat drastis dibanding baseline yang hanya mencari kata asli.
📋 Contoh Lapangan Menambah Cakupan
SQL LIKE di sini mencari tidak hanya pada deskripsi formal KBLI, tetapi juga pada field contoh_lapangan. Ini berarti query informal dari pengguna lapangan kini dapat dicocokkan dengan data aktivitas nyata yang menggunakan bahasa sehari-hari — mengurangi ketidakcocokan bahasa (lexical gap).
🔬 Perbandingan dengan Baseline
