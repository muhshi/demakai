📊 Perbandingan & Analisis

🔍 Dampak Preprocessing pada SQL LIKE
Preprocessing (expansion + sinonim) memperluas cakupan token yang dicari. Query seperti "TUKANG BATU" akan diperluas menjadi "tukang, batu, teknisi, pekerja, operator" sehingga lebih banyak entri KBLI/KBJI yang relevan dapat ditemukan meski kosakata formal berbeda. Namun SQL LIKE tetap terbatas pada pencocokan teks harfiah.
🧠 Dampak Preprocessing pada Hybrid
Preprocessing memberikan dua keuntungan sekaligus untuk Hybrid: (1) token yang sudah diperluas digunakan oleh SQL LIKE sehingga recall meningkat; (2) teks yang sudah di-clean digunakan sebagai input embedding Gemini, menghasilkan vektor yang lebih terfokus dan representasi semantik yang lebih akurat.
🏆 Metode Paling Optimal
SQL LIKE + Preprocessing memberikan performa terbaik berdasarkan MRR gabungan. Hybrid unggul karena dimensi ganda (lexical + semantic) mampu menjembatani gap antara bahasa informal query lapangan dan terminologi formal KBLI/KBJI yang tidak dapat dijangkau SQL LIKE saja.
